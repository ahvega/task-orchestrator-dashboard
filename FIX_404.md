# Fix: 404 Error on /api/projects/summary

## Problem
The new API endpoints return 404 because the server is running the old version of `server_v2.py` without the Phase 1 endpoints.

## Solution

**Restart the server to load the updated code:**

### Option 1: Using Quick Start Script (Recommended)

```powershell
# Stop the current server (Ctrl+C in its terminal)
# Or kill the process:
Get-Process python | Where-Object { $_.MainWindowTitle -like "*python*" -or $_.Path -like "*task-orchestrator*" } | Stop-Process

# Start fresh
.\start_server.ps1
```

### Option 2: Manual Restart

```powershell
# 1. Stop the server (Ctrl+C in the terminal running it)

# 2. Activate venv
.\venv\Scripts\Activate.ps1

# 3. Start server
python server_v2.py
```

## Verify It Works

After restarting, the server should show:
```
============================================================
Task Orchestrator Dashboard - Starting
============================================================
Using database: E:\MyDevTools\...\data\tasks.db
Database pool initialized successfully
Starting WebSocket database watcher...
Dashboard server ready!
============================================================
```

Then test the endpoint:
```powershell
# In a new terminal
Invoke-RestMethod -Uri "http://localhost:8888/api/projects/summary" | ConvertTo-Json
```

Should return:
```json
{
  "projects": [...],
  "count": N
}
```

## Why This Happened

Python loads the code when the server starts. Changes to `server_v2.py` require a restart to take effect. FastAPI's auto-reload feature may not be enabled in production mode.

## Quick Check if Server Has New Endpoints

Visit: http://localhost:8888/docs

You should see these endpoints in the API documentation:
- ✅ `/api/projects/summary`
- ✅ `/api/projects/{project_id}/overview`
- ✅ `/api/recent-activity`
- ✅ `/api/projects/most-recent`

If they're not there, the server needs to be restarted.
